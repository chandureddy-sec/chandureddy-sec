## Featured Projects

<details>
<summary><b>Network Traffic Filtering — Intrusion Detection System (NTF-IDS)</b></summary>

<br>

A real-world-grade **Network Traffic Filtering and Intrusion Detection System** built in Python, engineered to monitor, classify, and alert on suspicious network behavior at the packet level. The system combines rule-based filtering with behavioral anomaly detection to identify threats including port scans, DoS patterns, and protocol anomalies in live or captured traffic.

<br>

| Attribute | Details |
|:---|:---|
| **Stack** | Python, Scapy, Socket, Threading, CSV Logging |
| **Scale** | Real-time packet capture across multiple interfaces |
| **Performance** | Low-latency processing with multithreaded packet queuing |
| **Security** | Configurable rule engine with whitelist/blacklist support |
| **Impact** | Applicable in SOC environments, lab networks, and educational deployments |
| **Repository** | [Network-Traffic-Filtering-IDS](https://github.com/chandureddy-sec/Network-Traffic-Filtering-Intrusion-Detection-System-NTF-IDS-) |

The NTF-IDS was designed to bridge the gap between theoretical intrusion detection concepts and practical implementation. It leverages Python's `scapy` library for deep packet inspection, applies configurable detection rules against live traffic streams, and outputs structured logs for downstream SIEM ingestion or manual analysis. The architecture emphasizes modularity — detection rules, alert handlers, and interface bindings are each independently configurable, making the system extensible for research and production-lab use.

<br>
